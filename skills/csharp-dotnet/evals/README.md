# Evals — csharp-dotnet

These cases are read by a human or an LLM judge, not by an automated test runner. To run them, install the skill and feed each prompt to the agent: `should_trigger` cases should make the agent reach for `csharp-dotnet` (check the routing against each `why`), and `should_not_trigger` cases should route to the named sibling instead (`route_to`). For the `capability` case, give the agent the scenario, then score the generated code against every line in `must_include` — each bullet is pass/fail, and a strong result hits all of them (net10.0 TFM, NRT on, record DTO separate from the entity, async with CancellationToken, validation + ProblemDetails, OpenAPI wired, AsNoTracking with no N+1). It is a rubric, not an assertion library, so judge intent and substance over exact wording.
