# Evals — instagram-api

These cases are prompts scored against the skill's `description` by an LLM router. `should_trigger` prompts must route to `instagram-api`; `should_not_trigger` prompts must route to the named sibling (`tiktok-api`, `social-publisher`, `youtube-api`, `video-shorts`) and never here. The `capability` case is graded by hand against its `must_include` rubric on a real generated flow — feed the scenario to the agent, then check the output covers the container dance (`media_type=REELS` → poll `status_code` to `FINISHED` → `media_publish`), an insights pull that uses only valid (non-deprecated) metrics, the `content_publishing_limit` check, a pinned `GRAPH_VERSION`, and a written artifact under `02-DOCS/wiki/shortform/`. No automated runner ships with this skill: run it by feeding each prompt to the router, confirming routing, and scoring the capability rubric manually. `scripts/verify.sh` separately lints generated artifacts/scripts for deprecated metric tokens, the three publish steps, and a pinned version.
